Zibersegurtasuna V16 baliza konektatuetan

Sarrera + praktika (2h)

Helburua: arriskua ulertu eta defentsa-kontrolak balidatu.

Zer gertatzen da balizak abisu bat bidaltzen duenean?

  • Balizak gorabehera detektatzen du.
  • Abisua backend-era bidaltzen du.
  • Plataformak erabakitzen du: onartu edo baztertu.

Muturretik muturrerako fluxua

  1. V16 gailua
  2. Komunikazio-sarea
  3. Backend hartzailea
  4. Trafiko plataforma/zerbitzuak

Aktibo kritikoak

  • Gailuaren identitatea
  • Abisuaren osotasuna
  • Denbora-freskotasuna
  • Erabilgarritasuna
  • Trazabilitatea

Mehatxu nagusiak

  • Suplantazioa
  • Replay-a
  • Tartez kanpoko datuak
  • Ráfaga / API abusua

Defentsa-lehentasuna

  • Altua: replay-a, suplantazioa, ráfaga
  • Ertaina: timestamp/koordenatu baliogabeak

Backend-eko kontrolak

  • JSON eskema zorrotza
  • security.sent_at (denbora-leihoa)
  • security.nonce (anti-replay)
  • Rate limiting identitatez
  • Logging-a reason eremuarekin

Praktikan zer neurtuko dugu

  • Baliogabeen onarpena (0ra hurbildu)
  • Arrazoidun bazterketa teknikoa
  • Txostenerako trazabilitatea

Panela: /aginte-panela · JSON: /events/recent

Hondar-arriskua

  • Produkzio-mailako kontrol osagarriak falta dira
  • Hurrengo pausoak: sinadura, PKI, mTLS, SOC

Laborategirako trantsizioa

  1. Abisu legitimoa
  2. Replay-a eta baliogabeak
  3. Panelean/log-ean behatu
  4. Ebidentzia-txostena osatu

Tokiko sandbox-a edo sare isolatua soilik.
